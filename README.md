# Трекер задач

Внутренняя система управления задачами для команд до 35 человек с канбан-доской, ролевым доступом и уведомлениями.

## Демо

**Ссылка на приложение:** https://task-tracker-production-fc36.up.railway.app

**Логин:** `admin@tracker.ru` / **Пароль:** `admin123`

---

## Зачем нужен

Команды разработки, дизайна и маркетинга используют разные инструменты для управления задачами: Excel-таблицы, мессенджеры, стикеры. Это приводит к:

- Потере задач между инструментами
- Отсутствию видимости загрузки команды
- Забытым дедлайнам
- Дублированию задач

**Трекер задач** решает это — единое рабочее пространство со структурированными задачами, ролями и уведомлениями.

## Целевые пользователи

| Роль | Кто | Что делает в системе |
|------|-----|---------------------|
| **Администратор** | Руководитель / IT | Управление пользователями, сброс паролей, полный доступ |
| **Руководитель проекта** | PM / Тимлид | Управление проектами, распределение задач, контроль сроков |
| **Сотрудник** | Разработчик / Дизайнер | Выполнение задач, комментирование, загрузка вложений |

## Возможности

### Управление задачами
- Создание, редактирование, удаление (мягкое — с возможностью восстановления)
- Статусы: `todo` → `in_progress` → `in_review` → `done`
- Приоритеты: критический, высокий, средний, низкий
- Дедлайн с автоматическим определением просроченных задач
- Автоматическая генерация кода задачи (формат `CODE###`)

### Канбан-доска
- Drag-and-drop между колонками
- Фильтрация по статусу и приоритету
- Визуальное отображение загрузки

### Проекты
- CRUD-операции с проектами
- Управление участниками (добавление / удаление)
- Код проекта (уникальный)
- Архивирование проектов

### Уведомления
- Автоматические уведомления при назначении задачи
- Уведомления о комментариях
- Счётчик непрочитанных
- Отметка как прочитанное (по одному или все сразу)

### Команда
- Просмотр всех сотрудников
- Активные и просроченные задачи на каждого
- Профиль с редактированием данных

### Комментарии и вложения
- Комментарии к задачам (с редактированием и удалением)
- Загрузка файлов (изображения, документы)
- Просмотр и скачивание вложений

### Поиск
- Глобальный поиск по задачам, проектам и сотрудникам
- Результаты с фильтрацией по типу

### Дашборд
- Общая статистика (задачи, проекты, уведомления)
- Последние задачи
- Распределение задач по статусам

### Администрирование
- Управление пользователями (создание, редактирование, деактивация)
- Сброс паролей с генерацией временного
- Ролевой доступ (RBAC)

## API-эндпоинты

### Авторизация
| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/auth/login` | Вход в систему |
| GET | `/api/auth/me` | Текущий пользователь |
| POST | `/api/auth/change-password` | Смена пароля |

### Задачи
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/tasks` | Список задач (фильтры: status, priority, project_id, overdue) |
| GET | `/api/tasks/my` | Мои задачи |
| GET | `/api/tasks/overdue` | Просроченные задачи |
| POST | `/api/tasks` | Создание задачи |
| GET | `/api/tasks/:id` | Детали задачи |
| PUT | `/api/tasks/:id` | Обновление задачи |
| DELETE | `/api/tasks/:id` | Мягкое удаление |
| POST | `/api/tasks/:id/restore` | Восстановление |

### Проекты
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/projects` | Список проектов |
| POST | `/api/projects` | Создание проекта |
| GET | `/api/projects/:id` | Детали проекта |
| PUT | `/api/projects/:id` | Обновление проекта |
| POST | `/api/projects/:id/members` | Добавление участника |
| DELETE | `/api/projects/:id/members/:userId` | Удаление участника |

### Комментарии
| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/tasks/:id/comments` | Добавление комментария |
| PUT | `/api/comments/:id` | Редактирование комментария |
| DELETE | `/api/comments/:id` | Удаление комментария |

### Уведомления
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/notifications` | Список уведомлений |
| GET | `/api/notifications/unread-count` | Счётчик непрочитанных |
| POST | `/api/notifications/:id/read` | Отметить как прочитанное |
| POST | `/api/notifications/read-all` | Отметить все как прочитанные |

### Команда
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/team` | Список сотрудников с активными задачами |

### Поиск
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/search?q=` | Глобальный поиск |

### Дашборд
| Метод | Путь | Описание |

```bash
# Клонировать репозиторий
git clone https://github.com/samovarovao-ops/Task-Tracker.git

# Установить зависимости
cd backend

# Запустить сервер

# Открыть в браузере
# http://localhost:3001
```

## Деплой на Railway

1. Fork репозитория на GitHub
2. В Railway: **New Project** → **Deploy from GitHub repo**
3. Выбрать форкнутый репозиторий
4. Railway автоматически установит зависимости и запустит сервер
5. В **Settings** → **Networking** → **Generate Domain** получить публичный URL
