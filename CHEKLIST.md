# Чек-лист тестирования трекера задач

## 1. Авторизация и учётные записи

### Вход в систему
- [ ] Вход с корректными данными (admin@tracker.ru / admin123)
- [ ] Вход с некорректным паролем — сообщение об ошибке
- [ ] Вход с несуществующим email — сообщение об ошибке
- [ ] Показ/скрытие пароля на форме входа
- [ ] Токен сохраняется в localStorage после входа
- [ ] При обновлении страницы пользователь остаётся авторизованным
- [ ] Выход из системы (кнопка выхода)

### Блокировка аккаунта
- [ ] 10 неудачных попыток входа подряд → блокировка на 15 минут
- [ ] После блокировки вход невозможен, показывается время до разблокировки

### Смена пароля (первый вход)
- [ ] При входе с временным паролем (is_first_login) показывается модальное окно смены пароля
- [ ] Обязательная смена пароля перед доступом к системе
- [ ] Пароль: минимум 8 символов, хотя бы одна буква и одна цифра
- [ ] Совпадение нового пароля и подтверждения
- [ ] Неверный текущий пароль — ошибка

### Смена пароля (профиль)
- [ ] Смена пароля в разделе «Профиль» с текущим паролем
- [ ] После смены пароля можно войти с новым паролем

### Сброс пароля (админ)
- [ ] Админ может сбросить пароль любому пользователю
- [ ] После сброса генерируется временный пароль
- [ ] Пользователь обязан сменить пароль при следующем входе

---

## 2. Пользователи (RBAC)

### Администратор
- [ ] Видит всех пользователей в разделе «Администрирование»
- [ ] Создаёт нового пользователя — система генерирует временный пароль
- [ ] Временный пароль показывается ОДИН раз при создании
- [ ] Редактирует данные любого пользователя
- [ ] Меняет роль пользователей
- [ ] Деактивирует пользователей
- [ ] Сбрасывает пароли

### Руководитель проекта
- [ ] Видит участников своих проектов
- [ ] Управляет участниками проектов (добавление/удаление)
- [ ] Полный доступ к задачам в своих проектах
- [ ] НЕ может создавать проекты
- [ ] НЕ может менять роли пользователей

### Сотрудник
- [ ] Видит только проекты, в которых состоит
- [ ] Видит задачи только своих проектов
- [ ] Редактирует только НАЗНАЧЕННЫЕ ЕМУ задачи
- [ ] Создаёт задачи, ТОЛЬКО если в проекте включён флаг «Разрешить сотрудникам создавать задачи»
- [ ] НЕ может управлять пользователями
- [ ] НЕ может создавать проекты

---

## 3. Проекты

### Создание проекта
- [ ] Только администратор может создавать проекты
- [ ] Обязательные поля: Название, Код проекта, Руководитель
- [ ] Код проекта уникальный (регистронезависимый)
- [ ] Код проекта используется для формирования ID задач (CODE###)
- [ ] При создании руководитель автоматически добавляется как участник

### Редактирование проекта
- [ ] Админ и руководитель проекта могут редактировать
- [ ] Изменение названия, описания, статуса, дат
- [ ] Настройка флага «Разрешить сотрудникам создавать задачи»

### Статусы проекта
- [ ] Планируется → Активный → Приостановлен → Завершён → Архивный
- [ ] Архивация — проект доступен только для чтения

### Участники
- [ ] Добавление участников из списка пользователей
- [ ] Удаление участников
- [ ] Нельзя удалить руководителя проекта

### Страница проекта
- [ ] Вкладка «Доска» — канбан-доска проекта
- [ ] Вкладка «Список задач» — табличное представление
- [ ] Вкладка «Участники» — список участников
- [ ] Вкладка «Информация» — редактирование данных проекта

---

## 4. Задачи

### Создание задачи
- [ ] Обязательные поля: Проект, Название
- [ ] Автоматическое формирование ID: CODE### (например, NDF125)
- [ ] Установка статуса по умолчанию (Бэклог)
- [ ] Назначение исполнителя, приоритета, дедлайна
- [ ] Добавление тегов

### Редактирование задачи
- [ ] Изменение названия, описания
- [ ] Смена статуса, приоритета, исполнителя, дедлайна
- [ ] Все изменения логируются в истории

### Приоритеты
- [ ] Низкий (зелёный), Средний (жёлтый), Высокий (оранжевый), Критический (красный)
- [ ] Цветовые метки отображаются на карточках и в таблице

### Дедлайн
- [ ] Установка дедлайна (дата/время)
- [ ] Просроченные задачи выделяются визуально (красный цвет, иконка ⚠)
- [ ] Просрочка отображается и на доске, и в списке, и в карточке задачи

### Удаление задачи
- [ ] Мягкое удаление (перемещение в корзину на 30 дней)
- [ ] Восстановление из корзины
- [ ] Полное удаление — только администратор

---

## 5. Канбан-доска

### Колонки
- [ ] 5 колонок: Бэклог, Запланировано, В работе, На проверке, Выполнено
- [ ] Заголовок содержит название и счётчик задач
- [ ] Колонка «Выполнено» свёрнута по умолчанию
- [ ] Кнопка «Показать/Скрыть» для колонки «Выполнено»

### Карточка задачи на доске
- [ ] Отображается: название, ID (CODE###)
- [ ] Аватар исполнителя (или инициалы)
- [ ] Цветовая метка приоритета
- [ ] Дедлайн (просрочка выделена)
- [ ] Индикаторы комментариев и вложений

### Drag-and-drop
- [ ] Перетаскивание между колонками меняет статус задачи
- [ ] Задача помещается наверх при переносе в новую колонку
- [ ] Визуальное перемещение сохраняется (обновление UI)
- [ ] Ошибка изменения статуса — откат визуального перемещения

### Права на перетаскивание
- [ ] Исполнитель может перемещать свои задачи
- [ ] Руководитель проекта может перемещать любые задачи проекта
- [ ] Администратор может перемещать любые задачи
- [ ] Сотрудник НЕ может перемещать задачи, назначенные другим

### Фильтры и поиск
- [ ] Фильтр по исполнителю
- [ ] Фильтр по приоритету
- [ ] Фильтр по тегам
- [ ] Фильтр по дедлайну (диапазон)
- [ ] Фильтр «только просроченные»
- [ ] Поиск по названию/ID (подстрока, регистронезависимый)

### Пустые состояния
- [ ] Пустая колонка показывает подсказку

---

## 6. Список задач

### Таблица
- [ ] Столбцы: ID, Название, Статус, Исполнитель, Приоритет, Дедлайн
- [ ] Статус отображается цветной меткой
- [ ] Приоритет отображается цветной точкой
- [ ] Клик по строке открывает карточку задачи

### Сортировка
- [ ] По приоритету (Критический → Низкий)
- [ ] По дедлайну (ближайший первый)
- [ ] По дате обновления

---

## 7. Комментарии

- [ ] Добавление комментария к задаче
- [ ] Комментарий отображается с автором, аватаром, датой
- [ ] Редактирование своего комментария
- [ ] Удаление своего комментария
- [ ] Администратор может удалить любой комментарий
- [ ] Сотрудник НЕ может редактировать/удалять чужие комментарии
- [ ] При добавлении комментария исполнителю приходит уведомление

## 8. Вложения

### Загрузка
- [ ] Форматы: jpg, png, webp, pdf, docx, xlsx, pptx, txt
- [ ] Лимит: до 25 МБ на файл
- [ ] Лимит: до 200 МБ на задачу
- [ ] Превью для изображений
- [ ] Иконки/название для остальных форматов

### Управление
- [ ] Скачивание вложений
- [ ] Удаление вложений (автор, исполнитель, руководитель проекта, админ)

- [ ] Страница уведомлений со списком

### События уведомлений

- [ ] Изменение приоритета
- [ ] Изменение исполнителя
- [ ] Новый комментарий к задаче

- [ ] Доска — горизонтальная прокрутка колонок
- [ ] Карточка задачи — модальное окно на весь экран

- [ ] Время загрузки страниц ≤ 2 секунд
